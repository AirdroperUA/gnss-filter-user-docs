# Самостійна установка

> Де купити плату: [GPS Spoofing Filter](https://airdroper.org/products/gps-spoofing-filter)

Цей посібник описує прошивку GNSS-фільтра на чисту плату
STM32F401CC BlackPill за допомогою ліцензійного ключа, придбаного в магазині.

Кожний ліцензійний ключ активує **одну плату**. Прошивка унікально
прив'язана до апаратного ID вашої конкретної плати і не може бути скопійована на іншу.

---

## Що потрібно

| Елемент | Примітки |
|---------|----------|
| STM32F401CC BlackPill V2.0 плата | WeAct Studio або аналог |
| ST-Link V2 програматор | Оригінальний або клон, ~$3 |
| 4 з'єднувальні дроти | 3V3, GND, SWDIO, SWCLK |
| ПК з Windows та інтернетом | |
| Ліцензійний ключ | З вашої покупки (напр. `GF-XXXX-XXXX-XXXX`) |

### Програмне забезпечення

1. **STM32CubeProgrammer** (безкоштовно) — завантажте з
   [st.com](https://www.st.com/en/development-tools/stm32cubeprog.html)
   та встановіть. Після встановлення додаток знайде його автоматично.

2. **AirDroper GNSS Filter** додаток — завантажте з [gps.airdroper.org/download/app](https://gps.airdroper.org/download/app).
   Це графічний інструмент, який виконує все за вас.

## Крок 1 — Підключення ST-Link

Підключіть ST-Link до SWD-роз'єму BlackPill чотирма дротами:

| Пін ST-Link | Пін BlackPill |
|-------------|---------------|
| 3V3 | 3V3 |
| GND | GND |
| SWDIO | DIO (PA13) |
| SWCLK | CLK (PA14) |

Живлення BlackPill від ST-Link 3V3 (або USB, обидва варіанти працюють).

```
  ST-Link V2                   BlackPill
  ┌──────────┐                ┌──────────────┐
  │      3V3 ├────────────────┤ 3V3          │
  │      GND ├────────────────┤ GND          │
  │    SWDIO ├────────────────┤ DIO (PA13)   │
  │    SWCLK ├────────────────┤ CLK (PA14)   │
  └──────────┘                └──────────────┘
```

## Крок 2 — Активація та прошивка

1. Відкрийте додаток **AirDroper GNSS Filter**
2. Оберіть режим **Активація (ST-Link)**
3. Введіть ліцензійний ключ (напр. `GF-XXXX-XXXX-XXXX`)
4. Підключіть рівно **один** ST-Link V2. Якщо підключено декілька
   (лабораторний стенд із кількома адаптерами), додаток зупиниться і
   попросить відключити інші — це запобігає прошивці не тієї плати.
5. Натисніть **Старт** — додаток автоматично:
   - Підключиться до плати через ST-Link
   - Зчитає унікальний апаратний ID плати
   - Завантажить прошивку, створену для вашої плати
   - Прошиє завантажувач, додаток і налаштування
   - Увімкне захист від зчитування

Якщо ви закриєте вікно додатку під час прошивки, процес
`STM32_Programmer_CLI` буде завершено коректно — адаптер ST-Link не
залишиться заблокованим.

Ви маєте побачити приблизно таке:
```
Connecting to target via ST-Link...
Reading board UID...
Board UID: aabbccddeeff1122334455 (short: 12345678)
Contacting license server...
Firmware version: 1.5.5
  Bootloader: 47104 bytes
  Application: 73732 bytes
  Metadata: 148 bytes
Erasing flash...
Writing bootloader (0x08000000)...
Writing application (0x0800C000)...
Writing metadata (0x0801FC00)...
Setting readout protection (RDP Level 1)...

Provisioning complete!
```

> **Примітка:** Якщо плата вже була активована раніше, додаток автоматично
> знімає захист, стирає пам'ять і прошиває заново — жодних ручних кроків.

## Крок 3 — Перевірка

1. Від'єднайте ST-Link
2. Подайте живлення на плату через USB або бортову систему живлення
3. Підключіться до Mission Planner і перевірте повідомлення про старт фільтра
   у вкладці Messages (напр. `GNSS filter v1.5.5 UID=12345678`)

## Оновлення прошивки

Оновлення прошивки використовують **той самий ST-Link V2 адаптер**, що й
початкова активація. Оновлення через USB-C більше не підтримуються — піни
PA11/PA12 тепер зарезервовані під GPS UART (USART6) контролера польоту,
і завантажувач більше не з'являється у системі як USB CDC-пристрій.

> **НЕ підключайте нічого до USB-C роз'єму BlackPill — ніколи.**
> PA11/PA12 фізично спільні з USB D-/D+, і щойно USB-кабель подає сигнали
> хоста на ці лінії, він конфліктує з драйвером USART6 — польотний
> контролер показуватиме «GPS: No GPS», а EKF3 не зможе вирівнятися,
> навіть якщо фільтр повідомляє про fix. Плата не має робочого USB-шляху:
> живлення береться від +5V/+3V3 GPS-роз'єму FC (або з SWD-роз'єму під
> час першої прошивки), ніколи з USB-C кабелю. Для оновлення прошивки
> використовуйте ST-Link V2 через SWD — це єдиний підтримуваний шлях.

### Що потрібно

Те саме обладнання, що й при першій прошивці:

| Елемент | Примітки |
|---------|----------|
| ST-Link V2 програматор | Оригінальний або клон, ~$3 |
| 4 з'єднувальні дроти | 3V3, GND, SWCLK, SWDIO |

### Підключення (4-пінний SWD-роз'єм на платі ICWkey)

| Пін ST-Link | Пін ICWkey SWD |
|-------------|----------------|
| 3V3 (VCC) | 3V3 |
| GND | GND |
| SWCLK | A14 |
| SWDIO | A13 |

```
  ST-Link V2                   ICWkey (SWD-роз'єм)
  ┌──────────┐                ┌──────────────┐
  │  3V3 VCC ├────────────────┤ 3V3          │
  │      GND ├────────────────┤ GND          │
  │    SWCLK ├────────────────┤ A14          │
  │    SWDIO ├────────────────┤ A13          │
  └──────────┘                └──────────────┘
```

### Процедура оновлення (ST-Link V2 SWD)

1. Підключіть ST-Link до SWD-роз'єму ICWkey як показано вище
2. Відкрийте додаток **AirDroper GNSS Filter**
3. Оберіть режим **Оновлення (ST-Link)**
4. Введіть ліцензійний ключ
5. Натисніть **Старт**
6. Додаток підключиться через ST-Link, завантажить нову прошивку,
   адаптовану під вашу плату, і прошиє її (~30–60 секунд)
7. Плата автоматично перезавантажиться після завершення

> **Примітка:** RDP1 (захист від зчитування) знімається й застосовується
> повторно автоматично — жодних ручних кроків не потрібно.

### Відновлення (ST-Link V2 SWD)

Якщо плата не завантажується — наприклад після зникнення живлення під час
прошивки — використайте те саме підключення ST-Link V2, що й вище, та
оберіть у додатку режим **Відновлення (ST-Link)**. Процедура відновлення
повторно виконує повну активацію з вашим існуючим ліцензійним ключем.

Відновлення може попросити вас фізично перепідключити USB до **чотирьох
разів** поспіль (на кожному діалозі). Це обов'язково: option байти
STM32F4 фіксуються лише при реальному падінні VDD, а натискання кнопки
`Reset` цього не дає.

> **УВАГА — НЕ використовуйте STM32CubeProgrammer GUI для ручного
> зняття RDP, nWRP чи SPRMOD на BlackPill.** Його послідовність
> запису option байтів на F4 може повністю заблокувати чип у стані,
> де SPRMOD фіксується і відмовляється очищатися — жодне ПЗ (наше
> чи від ST) не зможе його відновити. **Завжди використовуйте кнопку
> "Відновити Плату" в цьому додатку** — вона виконує єдину перевірену
> послідовність для цих плат. Якщо "Відновити Плату" повідомить, що
> чип апаратно застряг після повного циклу перепідключень, замініть
> BlackPill (~5–4 дол.) — ваша ліцензія прив'язана до UID чипа,
> і нова плата провізіонується чисто з тим самим ключем.

## Журнал подій спуфінгу

**Плата фільтра не зберігає жодних логів спуфінгу.** Кожен DR1-перехід та
оновлення оцінки достовірності надсилаються в реальному часі як
MAVLink-повідомлення **STATUSTEXT** і **NAMED_VALUE_INT**, які ArduPilot
записує на SD-карту контролера польоту як стандартні dataflash-записи
(`MSG` і `NVLI`). SD-карта є єдиним джерелом істини для розслідування подій.

EEPROM-кільцевий буфер на платі вперше було припинено у v1.6.0 (тоді з
застосунку прибрали кнопку «Завантажити логи») і остаточно видалено з
прошивки у v1.6.8.

### Де знайти логи

1. Дістаньте SD-карту з контролера польоту (або завантажте `.bin`-лог
   через MAVFTP / Mission Planner)
2. Відкрийте `.bin`-лог у **Mission Planner** → *DataFlash Logs* → *Review a Log*
3. Події спуфінгу з'являються як рядки `MSG`, що починаються з `GNSS:` або
   `DR1:`, плюс записи `NVLI` з поточним балом достовірності `DR_CONF`
4. Опційно: перетягніть той самий `.bin` у [UAV Log Viewer](https://plot.ardupilot.org)
   для графічної шкали часу

> **Примітка щодо приватності:** якщо ви публічно ділитесь dataflash-логом
> FC (наприклад, для підтримки), GPS-координати запечені у власні записи
> ArduPilot `GPS` і `POS` — на них не впливає тунабл `LOG_LOC=0` фільтра,
> який редагує лише наші `STATUSTEXT`-рядки. Обріжте або очистьте `.bin`
> перед поширенням, якщо координати чутливі.

### Карта РЕБ перешкод

[Карта РЕБ перешкод](https://gps.airdroper.org/ew-map) — безкоштовна публічна карта глобальних GNSS перешкод у реальному часі з кількох джерел (авіаційні ADS-B, морські AIS, повітряні тривоги України, 61 відома зона РЕБ, краудсорсингові звіти та дані плат). Не потребує входу. Можна вбудувати на свій сайт через [gps.airdroper.org/ew-map/embed](https://gps.airdroper.org/ew-map/embed), підписатися на email/Telegram сповіщення або використовувати публічний API для оцінки ризику маршруту. Див. [FAQ](#faq) для деталей.

## Усунення неполадок

| Проблема | Рішення |
|----------|---------|
| "STM32_Programmer_CLI not found" | Встановіть STM32CubeProgrammer і перезапустіть додаток. Якщо не допомогло, додайте теку `bin` CubeProgrammer до системного PATH. |
| "Failed to connect" | Перевірте підключення ST-Link (3V3, GND, SWDIO, SWCLK). Спробуйте інший USB-порт. |
| "Invalid license key" | Перевірте ключ з листа про покупку |
| "License already activated on a different board" | Кожний ключ працює лише на одній платі. Зверніться до підтримки для заміни. |
| "ST-Link not found" (оновлення) | Підключіть ST-Link до іншого USB-порту й перевірте 4-пінне SWD-підключення (3V3, GND, A14/SWCLK, A13/SWDIO). |
| Попередження RDP Level 1 при повторній прошивці | Плата вже захищена. Додаток обробляє це автоматично. |


### Початкова активація (ST-Link)

```
gnss-provision activate --license GF-XXXX-XXXX-XXXX --server https://gps.airdroper.org
```

### Оновлення прошивки (ST-Link)

```
gnss-provision update --license GF-XXXX-XXXX-XXXX --uid <ваш-24-символьний-uid> --stlink --server https://gps.airdroper.org
```

UID — це повний 24-символьний hex-рядок, показаний під час початкової
активації. Перед запуском команди підключіть ST-Link до SWD-роз'єму
(3V3, GND, A14/SWCLK, A13/SWDIO).
