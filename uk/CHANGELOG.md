# Журнал змін

Всі важливі зміни прошивки та інструментів AirDroper GNSS Spoofing Filter
задокументовані нижче.

---

## v1.5.0 — SpoofAnalytics (2026-04-03)

### Прошивка
- Журналювання подій спуфінгу у флеш-пам'ять плати (EEPROM сектор 5,
  кільцевий буфер, максимум 330 подій)
- Кожна подія містить: час, GPS-координати, висоту, кількість супутників,
  HDOP, SNR min/max, причину виявлення, тривалість
- Нові команди Phase-C: `LOG_READ` (0x06) та `LOG_CLEAR` (0x07) для
  зчитування/очищення журналів через UART

### Сервер ліцензій
- `POST /api/v1/upload-logs` — прийом журналів подій спуфінгу з плат
- `GET /api/v1/dashboard/events` — пагіновані події за ліцензійним ключем
- `GET /api/v1/dashboard/summary` — зведена статистика за ліцензійним ключем
- `GET /dashboard` — інтерактивна хмарна панель з картою Leaflet,
  хронологією подій та діаграмами причин
- Створення ліцензій тепер генерує випадкові ключі з префіксу
  (не потрібно вводити ключ вручну)
- Валідація вводу: формат UID hex, ліміт кількості подій, обмеження пагінації
- Захист від XSS на сторінках адміністратора та панелі

### Інструмент провізіонування (GUI)
- Кнопка "Завантажити логи (UART)" — зчитує події з плати, завантажує на
  хмарну панель, очищує журнал на платі
- Для завантаження логів потрібен ліцензійний ключ (захист даних польоту
  якщо плату знайде стороння особа)
- Посилання "Переглянути панель" після успішного завантаження логів

### Документація
- Новий розділ у посібнику з установки: журнал подій спуфінгу та хмарна панель
- Українській переклад нових розділів

## v1.4.3 — ParamSaveFix (2026-03-09)

### Прошивка
- Виправлено помилку збереження/відновлення параметрів EEPROM, через яку
  значення тюнінгу могли втрачатися після перезавантаження
- Змінено час блокування DR1 за замовчуванням на 120 секунд
- Виправлено типи повідомлень MAVLink для коректного відображення в Mission Planner
- Стабільна збірка, протестована на квадрокоптері

### Система ліцензування (нова)
- Підпис прошивки Ed25519 з Monocypher
- Потокове шифрування BLAKE2b — ключі для кожного пристрою на основі UID плати
- Протокол зашифрованого оновлення Phase-C через UART (HELLO/BEGIN/DATA/END/ABORT)
- Завантажувач з RDP Level 1, перевіркою підпису та встановленням зашифрованої прошивки
- Сервер ліцензій FastAPI з SQLite
- Бінарний патчинг шаблону завантажувача (без перекомпіляції для кожної плати)
- GUI провізіонування (tkinter) з активацією ST-Link та оновленням через UART
- Пакування в один .exe для Windows через PyInstaller
- CI пайплайн: збірки прошивки + завантажувача, набір тестів Python
- Розгортання сервера: DigitalOcean, nginx, Let's Encrypt, systemd

### Документація
- Повний набір документації (EN + UK): підключення, налаштування, робота,
  тюнінг, шпаргалка, лабораторна валідація, конфігурація приймача, установка, FAQ
- Документація для розробників: CI/CD, оновлення прошивки, архітектура
  ліцензування, розгортання сервера

## v1.4.2 — UM980Support (2026-03-24)

### Прошивка
- Додано підтримку приймача Unicorecomm UM980 поряд з u-blox
- Параметр `GNSS_TYPE`: 0 = u-blox (за замовчуванням), 1 = UM980
- Парсинг бінарного протоколу UM980 (BESTNAV, BESTSAT)
- Послідовність автоконфігурації UM980
- Налаштування швидкості передачі для UM980 (460800 за замовчуванням)

### Документація
- Посібник з конфігурації приймача (`09_receiver_config.md`) з профілями
  команд UM980 та u-blox
- Вимога ArduPilot `GPS_AUTO_CONFIG=0` для UM980
