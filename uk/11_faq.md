## Загальне

### Що робить цей фільтр?

Він розташовується між GNSS-приймачем і контролером польоту. Фільтр у
реальному часі моніторить якість GPS і блокує передачу GNSS-даних на FC,
коли виявляє аномалії — спуфінг, глушіння або втрату сигналу. FC тоді
переходить на dead-reckoning (інерційну навігацію), поки не повернеться
чистий GPS-сигнал. Див. [Огляд пристрою](#device-overview) для детального пояснення.

### Які контролери польоту підтримуються?

Будь-який FC на базі ArduPilot з версією **ArduPilot 4.6.1 або новішою** (Copter, Plane,
Rover). Фільтр комунікує через стандартну MAVLink2-телеметрію.

### Які GNSS-приймачі працюють?

- **u-blox**: сімейства M8, M9, M10, F9, F10 (`GNSS_TYPE=0`)
- **Unicore**: UM980, UM981, UM982 (`GNSS_TYPE=1`) — потребує одноразового налаштування, див. [Конфігурація приймача](#receiver-config)

### Чи можна використовувати з PX4 або iNav?

Наразі ні. MAVLink-інтеграція фільтра розроблена для ArduPilot.

### Від яких систем РЕБ захищає?

Від усіх. Фільтр виявляє спуфінг і глушіння GPS за аномаліями сигналу
(стрибки позиції, патерни SNR, розходження висоти, зменшення кількості
супутників), а не за ідентифікацією конкретного обладнання РЕБ.

**Протестовано/розроблено проти:**

- **Українські:** Ліма, Пательня, Покрова (GPS спуфінг); Буковель, Нота, Дамба, Анклав (глушіння GNSS); Кульбаба, PARASOL, Піранья AVD 360 (придушення дронів)
- **Російські:** Поле-21/Field-21, Шипівник-Аеро (GPS спуфінг); Р-330Ж Житель (глушіння+спуфінг GNSS); Красуха-2/4 (глушіння радарів); Борисоглібськ-2, Леєр-3, Мурманськ-БН, Палантин (глушіння зв'язку); Репеллент-1, Інфауна (придушення дронів/СВП)
- **Білоруські:** Гроза (глушіння GNSS)

Також працює проти будь-яких майбутніх систем РЕБ, що впливають на
сигнали GNSS. Див. [Огляд пристрою — Сумісність із системами РЕБ](#device-overview)
для повної таблиці з частотами та можливостями.

## Апаратне забезпечення

### Яка плата потрібна?

STM32F401CC BlackPill V2.0 (WeAct Studio). Доступна на AliExpress,
Amazon або в магазинах електроніки приблизно за ~$5. Можна [придбати готову плату](https://airdroper.org/products/gps-spoofing-filter) або прошити самостійно — див. [Самостійна установка](#self-install).

### Чи можна використати іншу плату STM32?

Ні. Прошивка скомпільована спеціально для розпіновки та карти пам'яті
STM32F401CC BlackPill.

### Які швидкості передачі використовуються?

| Канал | Швидкість |
|-------|-----------|
| GNSS ↔ STM32 | 460800 (за замовчуванням, автовизначення) |
| FC MAVLink ↔ STM32 | 115200 |
| FC GPS ← STM32 | 460800 |

### Чи потрібно окреме живлення для STM32?

Ні. Він отримує живлення від телеметричного UART FC або від USB. Переконайтесь,
що всі пристрої мають спільну землю. Див. [Схема підключення](#wiring) для деталей.

## Налаштування

### FC показує "No GPS" або "GPS: not healthy"

1. Перевірте підключення — TX і RX мають бути перехрещені (див. [Схема підключення](#wiring))
2. Переконайтесь, що `SERIAL_BAUD` відповідає (460800 для GPS UART, 115200 для MAVLink)
3. Тимчасово встановіть `FCGPS_FWD=1`, щоб обійти DR1 і переконатись, що дані доходять до FC
4. Для UM980: переконайтесь, що `GPS_AUTO_CONFIG=0` в ArduPilot (див. [Конфігурація приймача](#receiver-config))

Якщо проблема залишається, див. [Діагностика підключення](#wiring-debug) для покрокового усунення несправностей.

### Як дізнатись, який GNSS_TYPE використовувати?

- Якщо ваш приймач u-blox (NEO-M8, NEO-M9, ZED-F9P тощо): `GNSS_TYPE=0`
- Якщо ваш приймач Unicore UM980, UM981 або UM982: `GNSS_TYPE=1`

Див. [Налаштування та прошивка](#setup-flash) для зміни цього параметра.

### Чи потрібно налаштовувати приймач UM980?

Так. STM32 **не** виконує автоконфігурацію UM980. Вам потрібно застосувати
профіль конфігурації через UPrecise перед першим польотом. Див.
[Конфігурація приймача](#receiver-config) для точних команд.

### Чи потрібно налаштовувати приймач u-blox?

Ні. STM32 автоматично налаштовує u-blox приймачі при завантаженні. Просто підключіть.

## Робота

### Що таке DR0 і DR1?

- **DR0** = штатний режим. GPS-дані передаються на контролер польоту.
- **DR1** = захисний режим. GPS-дані заблоковані. FC використовує dead-reckoning.

Див. [Огляд пристрою](#device-overview) для детального пояснення.

### Як швидко спрацьовує DR1?

Залежить від типу тригера:
- **Немає фіксу / мало супутників**: одразу
- **Стрибок позиції**: одразу при виявленні
- **EKF-тригер**: налаштовувана затримка через `EKF_TRIPMS` (за замовчуванням 0 = одразу)
- **SNR-аномалія**: після утримання `SNR_HOLDMS`

Див. [Тюнінг](#tuning) для налаштування цих порогів.

### Як літак повертається до GPS після DR1?

Фільтр перевіряє:
1. Кількість супутників >= `RJ_MIN_SATS`
2. HDOP <= `RJ_MAX_HD`
3. Умови виконуються протягом `RJ_STAB_MS`
4. Минув мінімальний час DR1 `DR_LOCK_MS`
5. (Опціонально) EKF валідний протягом `EKF_OKRJMS`

Коли всі умови виконані, відновлюється DR0 і GPS-дані знову передаються. Див. [Тюнінг](#tuning) для деталей кожного параметра.

### Що відбувається, якщо літак втрачає GPS надовго?

FC використовує dead-reckoning (IMU + компас + датчик швидкості). DR накопичує
дрейф з часом, тому літак має бути налаштований на повернення додому або
кружляння при втраті GPS. Фільтр має watchdog відновлення GNSS, який
надсилає команди гарячого/холодного перезапуску приймачу.

### Чи можна бачити, що робить фільтр під час польоту?

Так. Фільтр надсилає статусні повідомлення через MAVLink кожні 10 секунд
(налаштовується через `LOG_MS`). У вкладці Messages Mission Planner ви побачите:
- Режим/стан: `ARM=... DR=... BLEND=... LAT=... LONG=...`
- Зведення GNSS: `data=... fix=... nav=... SATS=... SNR=...`

Див. [Швидка довідка](#cheat-sheet) для швидкого пояснення цих повідомлень.

### Що робить пін B5?

Він видає імпульс високого рівня тривалістю 3 секунди при кожному переході DR0→DR1. Ви можете
підключити до нього світлодіод, зумер або зовнішній логер для фізичної
індикації активації захисного режиму. Див. [Схема підключення](#wiring) для деталей підключення.

### Що таке оцінка достовірності спуфінгу (DR_CONF)?

Починаючи з v1.5.5, фільтр обчислює бал 0–100 з до 8 незалежних
сигналів виявлення. Вищий бал = більше ознак спуфінгу. Бал передається
як `DR_CONF` у телеметрії MAVLink і записується в кожен лог подій.

u-blox використовує всі 8 сигналів. UM980 використовує доступні через
NMEA (~5 з 8). Бал автоматично адаптується — недоступні сигнали
виключаються зі зваженого середнього.

### Чи всі функції виявлення працюють з UM980?

Більшість функцій працюють з обома приймачами. Три сигнали **тільки для u-blox**:

- **Аналіз pseudorange residual** (з NAV-SAT)
- **Виявлення різкої зміни GDOP** (з NAV-DOP)
- **Виявлення стрибку тактового зсуву** (з NAV-CLOCK)

Основний захист (стрибок позиції, висота, SNR, курс, час, гео-огорожа)
працює з обома приймачами.

### Що таке DR1_MAXMS та FENCE_RAD?

- **DR1_MAXMS**: Примусовий вихід з DR1 після вказаної кількості мілісекунд.
  `0` = вимкнено. Корисно для місій, де нескінченне блокування GPS гірше за невизначений GPS.
- **FENCE_RAD**: Тригер DR1 при виході позиції за радіус від першого фіксу
  (макс. 2 000 000 м = 2000 км). За замовчуванням `600000` (600 км).
  `0` = вимкнено. Виявляє повільний дрейф позиції при спуфінгу.

### Чи можна експортувати події в Google Earth?

Так (v1.5.5+). Хмарна панель надає посилання для експорту KML та GPX.
KML можна відкрити в Google Earth для перегляду подій на 3D-карті.

## Специфіка UM980

### UM980 показує червону швидкість/позицію в Mission Planner

Найімовірніші причини:
1. **`GPS_AUTO_CONFIG` увімкнений** (за замовчуванням) — ArduPilot перезаписує вашу
   конфігурацію UM980 на `MODE MOVINGBASE`, що некоректно для одноантенного
   БПЛА. Встановіть `GPS_AUTO_CONFIG=0`.
2. **Відсутній `MODE ROVER UAV`** у конфігурації UM980 — застосуйте повний профіль
   з [Конфігурації приймача](#receiver-config).

### Які повідомлення ArduPilot потребує від UM980?

Лише три: **GNGGA**, **GNRMC** та **AGRICA** — всі на частоті 5 Гц (період 0.2).
Драйвер не аналізує GPGSA, GPGSV, GPGST або будь-які бінарні повідомлення.

### Чи потрібно налаштовувати UM980 при кожному увімкненні?

Ні. Команда `SAVECONFIG` наприкінці профілю UPrecise зберігає
все до енергонезалежної флеш-пам'яті UM980. Налаштуйте один раз — літайте завжди.

## Ліцензування та оновлення

### Чи можна встановити прошивку самостійно?

Так. Придбайте ліцензійний ключ у [магазині](https://airdroper.org/products/gps-spoofing-filter) та слідуйте [Самостійній установці](#self-install). Усі операції — початкова активація, оновлення прошивки та відновлення — використовують **той самий ST-Link V2 адаптер (~$3)**, підключений до 4-пінного SWD-роз'єму (3V3, GND, A14/SWCLK, A13/SWDIO). Плата більше не приймає оновлення прошивки через USB-C.

### Чи можна використовувати ліцензію на кількох платах?

Ні. Кожний ліцензійний ключ активує **одну плату**. Прошивка прив'язана до
унікального апаратного ID цієї плати.

### Як оновити прошивку?

Підключіть ST-Link V2 до 4-пінного SWD-роз'єму (3V3 → 3V3, GND → GND, SWCLK → A14, SWDIO → A13), відкрийте додаток **AirDroper GNSS Filter**, оберіть **Оновлення (ST-Link)**, введіть ліцензійний ключ і натисніть Старт. Додаток автоматично знімає RDP, прошиває плату та повторно вмикає захист. Див. розділ оновлення в [Самостійній установці](#self-install).

### Чи можна перепровізіонувати плату, яка вже має прошивку?

Якщо плата вже має RDP1 (захист від зчитування), спочатку потрібно його зняти — це стирає flash. Підключіть **ST-Link адаптер** ($3-5) через SWD — інструмент прошивки зніме RDP автоматично.

### Як завантажити логи спуфінгу?

Логи спуфінгу більше не вивантажуються з плати фільтра. Починаючи з прошивки v1.6.0, кожна подія виявлення надсилається як MAVLink-повідомлення **STATUSTEXT** і **NAMED_VALUE_INT**, які ArduPilot записує на SD-карту контролера польоту як стандартні dataflash-записи `MSG` і `NVLI`. Дістаньте `.bin`-лог із SD-карти (або завантажте його через MAVFTP / Mission Planner) і перегляньте у Mission Planner чи [UAV Log Viewer](https://plot.ardupilot.org). Див. розділ логів у [Самостійній установці](#self-install).

### Що робити, якщо плата вийшла з ладу?

Зверніться до служби підтримки. Ми можемо деактивувати UID старої плати, щоб ви могли
використати ту саму ліцензію на новій платі.

### Чи може хтось вкрасти мою прошивку?

Флеш-пам'ять захищена RDP Level 1 — спроба зчитування через SWD ініціює
mass erase. Прошивка також зашифрована унікальним ключем плати, тому навіть
якщо її витягти, вона не працюватиме на іншій платі.

## Карта РЕБ перешкод

### Що таке карта РЕБ перешкод?

Це жива веб-карта на [gps.airdroper.org/ew-map](https://gps.airdroper.org/ew-map), яка показує глобальні GNSS перешкоди в реальному часі. Вона об'єднує дані з кількох джерел:

- **Авіаційні ADS-B дані** (OpenSky) — виявляє перешкоди порівнянням барометричної та GPS висоти по 5 регіонах світу
- **Морські AIS дані** — виявляє судна з збоями GPS у Балтиці та за її межами
- **Повітряні тривоги України** — статус тривог по областях у реальному часі
- **61 відома зона РЕБ** — статичні військові зони перешкод по всьому світу з перехресним підтвердженням живими даними
- **Дані плат** — події спуфінгу з розгорнутих фільтрів GNSS
- **Краудсорсингові звіти** — повідомлення користувачів про перешкоди

Карта безкоштовна і не потребує входу в систему.

### Чи можна вбудувати карту РЕБ на свій сайт?

Так. Використовуйте віджет `gps.airdroper.org/ew-map/embed` в iframe:
```html
<iframe src="https://gps.airdroper.org/ew-map/embed" width="100%" height="500"></iframe>
```

### Чи можна отримувати сповіщення про перешкоди?

Так, три способи:
1. **Email/webhook** — підпишіться внизу сторінки карти РЕБ або через `POST /api/v1/ew-map/subscribe`
2. **Telegram** — напишіть боту команду `/subscribe`
3. **Публічний API** — зареєструйте безкоштовний API-ключ через `POST /api/v1/ew-map/register-key` (100 запитів/день)

### Чи можна перевірити ризик перешкод для маршруту?

Так. Використовуйте API оцінки ризику маршруту:
```
POST https://gps.airdroper.org/api/v1/ew-map/route-check
Body: {"waypoints": [[lat1, lon1], [lat2, lon2], ...]}
```
Повертає оцінку ризику (0–100) для кожного сегменту на основі відомих зон, живих ADS-B даних та повітряних тривог.

### Чи прогнозує карта майбутні перешкоди?

Так. Сервер будує патерни ймовірності за часом доби на основі 7 днів історичних даних. Запитуйте прогнози через `GET /api/v1/ew-map/predictions/now`.
