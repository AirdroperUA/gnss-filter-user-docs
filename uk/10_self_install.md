# Самостійна установка

> Де купити плату: [GPS Spoofing Filter](https://airdroper.org/product/gps-spoofing-filter/)

Цей посібник описує прошивку GNSS-фільтра на чисту плату
STM32F401CC BlackPill за допомогою ліцензійного ключа, придбаного в магазині.

Кожний ліцензійний ключ активує **одну плату**. Прошивка унікально
прив'язана до апаратного ID вашої конкретної плати і не може бути скопійована на іншу.

---

## Що потрібно

| Елемент | Примітки |
|---------|----------|
| STM32F401CC BlackPill V2.0 плата | WeAct Studio або аналог |
| ST-Link V2 програматор | Оригінальний або клон, ~$3 |
| 4 з'єднувальні дроти | 3V3, GND, SWDIO, SWCLK |
| ПК з Windows та інтернетом | |
| Ліцензійний ключ | З вашої покупки (напр. `GF-XXXX-XXXX-XXXX`) |

### Програмне забезпечення

1. **STM32CubeProgrammer** (безкоштовно) — завантажте з
   [st.com](https://www.st.com/en/development-tools/stm32cubeprog.html)
   та встановіть. Після встановлення додаток знайде його автоматично.

2. **AirDroper GNSS Filter** додаток — завантажте з [gps.airdroper.org/download/app](https://gps.airdroper.org/download/app).
   Це графічний інструмент, який виконує все за вас.

## Крок 1 — Підключення ST-Link

Підключіть ST-Link до SWD-роз'єму BlackPill чотирма дротами:

| Пін ST-Link | Пін BlackPill |
|-------------|---------------|
| 3V3 | 3V3 |
| GND | GND |
| SWDIO | DIO (PA13) |
| SWCLK | CLK (PA14) |

Живлення BlackPill від ST-Link 3V3 (або USB, обидва варіанти працюють).

```
  ST-Link V2                   BlackPill
  ┌──────────┐                ┌──────────────┐
  │      3V3 ├────────────────┤ 3V3          │
  │      GND ├────────────────┤ GND          │
  │    SWDIO ├────────────────┤ DIO (PA13)   │
  │    SWCLK ├────────────────┤ CLK (PA14)   │
  └──────────┘                └──────────────┘
```

## Крок 2 — Активація та прошивка

1. Відкрийте додаток **AirDroper GNSS Filter**
2. Оберіть режим **Активація (ST-Link)**
3. Введіть ліцензійний ключ (напр. `GF-XXXX-XXXX-XXXX`)
4. Натисніть **Старт** — додаток автоматично:
   - Підключиться до плати через ST-Link
   - Зчитає унікальний апаратний ID плати
   - Завантажить прошивку, створену для вашої плати
   - Прошиє завантажувач, додаток і налаштування
   - Увімкне захист від зчитування

Ви маєте побачити приблизно таке:
```
Connecting to target via ST-Link...
Reading board UID...
Board UID: aabbccddeeff1122334455 (short: 12345678)
Contacting license server...
Firmware version: 1.5.5
  Bootloader: 47104 bytes
  Application: 73732 bytes
  Metadata: 148 bytes
Erasing flash...
Writing bootloader (0x08000000)...
Writing application (0x0800C000)...
Writing metadata (0x0801FC00)...
Setting readout protection (RDP Level 1)...

Provisioning complete!
```

> **Примітка:** Якщо плата вже була активована раніше, додаток автоматично
> знімає захист, стирає пам'ять і прошиває заново — жодних ручних кроків.

## Крок 3 — Перевірка

1. Від'єднайте ST-Link
2. Подайте живлення на плату через USB або бортову систему живлення
3. Підключіться до Mission Planner і перевірте повідомлення про старт фільтра
   у вкладці Messages (напр. `GNSS filter v1.5.5 UID=12345678`)

## Оновлення прошивки

Після початкової прошивки оновлення **не** потребують ST-Link.
Найпростіший варіант — **USB-C** (не потрібно додаткове обладнання). Також можна використати USB-UART адаптер.

### Варіант А — USB-C (найпростіший)

1. Підключіть BlackPill до ПК кабелем USB-C
2. Відкрийте додаток **AirDroper GNSS Filter**
3. Оберіть режим **Оновлення**
4. Встановіть Порт на **(USB-C авто)**
5. Введіть ліцензійний ключ
6. Натисніть **Старт**
7. Коли з'явиться запит, **натисніть кнопку reset** на BlackPill
8. Додаток знайде плату автоматично і прошиє нову прошивку (~30-60 секунд)

### Варіант Б — USB-UART адаптер

#### Що потрібно

| Елемент | Примітки |
|---------|----------|
| USB-UART адаптер | CP2102, CH340, або FTDI, ~$2 |
| 3 з'єднувальні дроти | TX, RX, GND |

### Підключення

| USB-UART адаптер | Пін BlackPill |
|------------------|---------------|
| TX | PA10 (RX) |
| RX | PA9 (TX) |
| GND | GND |

**Важливо:** Від'єднайте контролер польоту від PA9/PA10 під час оновлення.
Це ті самі піни, що використовуються для телеметрії контролера польоту.

```
  USB-UART адаптер             BlackPill
  ┌──────────┐                ┌──────────────┐
  │       TX ├────────────────┤ PA10 (RX)    │
  │       RX ├────────────────┤ PA9  (TX)    │
  │      GND ├────────────────┤ GND          │
  └──────────┘                └──────────────┘
     (від'єднайте контролер польоту від PA9/PA10)
```

#### Процедура оновлення (UART)

1. Підключіть USB-UART адаптер як показано вище
2. Відкрийте додаток **AirDroper GNSS Filter**
3. Оберіть режим **Оновлення (UART)**
4. Оберіть COM-порт вашого адаптера (визначається автоматично)
5. Введіть ліцензійний ключ
6. Натисніть **Старт**
7. Коли з'явиться запит, **натисніть кнопку reset** на BlackPill
8. Додаток прошиє нову прошивку (~30-60 секунд)
9. Плата автоматично перезавантажиться після завершення

## Журнал подій спуфінгу та хмарна панель

Починаючи з прошивки v1.5.0 фільтр записує кожну подію спуфінгу у
пам'ять плати. Кожна подія містить час, GPS-координати, кількість
супутників, силу сигналу та причину виявлення.

### Завантаження журналів

**Варіант А — USB-C (найпростіший, без додаткового обладнання):**

1. Підключіть BlackPill до ПК кабелем USB-C
2. У додатку встановіть Порт на **(USB-C авто)**
3. Введіть ліцензійний ключ та UID
4. Натисніть **Завантажити логи** і натисніть **кнопку reset** на BlackPill коли запитає
5. Додаток знайде плату автоматично, завантажить події, відправить на хмарну панель і очистить журнал

**Варіант Б — USB-UART адаптер:**

1. Підключіть USB-UART адаптер (таке ж підключення як для оновлення)
2. У додатку оберіть COM-порт адаптера
3. Введіть ліцензійний ключ та UID
4. Натисніть **Завантажити логи** і натисніть **reset** коли запитає

Ліцензійний ключ потрібен для завантаження журналів — якщо хтось знайде
розбитий дрон, він не зможе отримати дані польоту без ключа.

### Хмарна панель

Відвідайте [gnss-filter.online/dashboard](https://gnss-filter.online/dashboard) і введіть ліцензійний
ключ для перегляду:

- **Карта подій** — всі події спуфінгу на карті з кольоровими маркерами
  за причиною виявлення
- **Статистика** — загальна кількість подій, кількість плат, розподіл причин
- **Список подій** — хронологія з координатами, висотою, кількістю
  супутників та даними сигналу
- **Оцінка достовірності** — кожна подія показує бал 0–100 з
  мульти-сигнального аналізу фільтра (v1.5.5+)
- **Експорт KML/GPX** — завантаження подій як KML (Google Earth) або
  GPX файлів для офлайн-аналізу (v1.5.5+)

Починаючи з v1.5.5, кожна подія також записує UTC час (година/хвилина з
GPS) та оцінку достовірності спуфінгу.

Панель доступна з будь-якого браузера. Кожний ліцензійний ключ бачить
лише свої дані.

## Усунення неполадок

| Проблема | Рішення |
|----------|---------|
| "STM32_Programmer_CLI not found" | Встановіть STM32CubeProgrammer і перезапустіть додаток. Якщо не допомогло, додайте теку `bin` CubeProgrammer до системного PATH. |
| "Failed to connect" | Перевірте підключення ST-Link (3V3, GND, SWDIO, SWCLK). Спробуйте інший USB-порт. |
| "Invalid license key" | Перевірте ключ з листа про покупку |
| "License already activated on a different board" | Кожний ключ працює лише на одній платі. Зверніться до підтримки для заміни. |
| "Timeout: no response from bootloader" (оновлення) | Натисніть кнопку reset на BlackPill, поки додаток очікує |
| Попередження RDP Level 1 при повторній прошивці | Плата вже захищена. Додаток обробляє це автоматично. |

<summary><strong>Просунуте: Альтернатива через командний рядок</strong></summary>

Якщо ви надаєте перевагу командному рядку замість графічного додатку, можна використати CLI-інструмент.

### Вимоги

Потрібен Python 3.10+ з пакетами:
```
pip install requests pyserial
```

### Початкова активація (ST-Link)

```
gnss-provision activate --license GF-XXXX-XXXX-XXXX --server https://gps.airdroper.org
```

### Оновлення прошивки (UART)

```
gnss-provision update --license GF-XXXX-XXXX-XXXX --uid <ваш-24-символьний-uid> --uart COM5 --server https://gps.airdroper.org
```

Замініть `COM5` на порт вашого адаптера. UID — це повний 24-символьний
hex-рядок, показаний під час початкової активації.

</details>
