# Швидка довідка (польові перевірки)

Це односторінкова довідка для швидких перевірок, режимів DR та типових дій.

## 1) Підсумок UART

- GNSS і STM32: `A2/A3`
- FC MAVLink і STM32: `A9/A10` (MAVLink2 @ 115200)
- FC GPS і STM32: `A11/A12` (зазвичай 460800)

## 2) Режим приймача (`GNSS_TYPE`)

- `GNSS_TYPE=0`: режим u-blox/UBX.
- `GNSS_TYPE=1`: режим UM980/UM981 NMEA.
- Після зміни `GNSS_TYPE` перезавантажте STM32 (`NRST` або повне вимкнення/увімкнення).
- Для кастомного u-blox без автоконфігу:
  - задайте `UBX_BAUD` як baud приймача (`0` означає автоконфіг ON),
  - перезавантажте STM32 для застосування.

## 3) Режими DR

- **DR0**: штатний режим, пересилання увімкнене.
- **DR1**: захисний режим, пересилання заблоковане.
- **B5**: високий імпульс (~3 с) на кожен перехід DR0 -> DR1.

## 4) Швидка діагностика

Якщо FC показує **No GPS config data**:

- Перевірте протокол і baud GPS UART на FC.
- Перевірте перехресне TX/RX підключення `A11/A12`.
- Перевірте спільну землю.

Якщо DR1 не вимикається:

- Перевірте no-fix / низьку кількість супутників.
- Перевірте пороги захистів (`ARM_*`, `SP_*`, `ALT_*`, `SNR_*`).
- Збільшіть `BOOT_DLYMS`, якщо хибний DR1 з'являється одразу після старту.

Якщо FC постійно показує **No Fix**:

- Перевірте, чи AUX-логіка RC не примусово вимикає GPS у налаштуваннях FC.

## 5) Команди `tune_cli.py`

Список параметрів:

```bash
py -3 .\tools\tune_cli.py --port COM12 --baud 115200 list
```

Читання одного параметра:

```bash
py -3 .\tools\tune_cli.py --port COM12 --baud 115200 get BLEND_MS
```

Зміна одного параметра:

```bash
py -3 .\tools\tune_cli.py --port COM12 --baud 115200 set RJ_BASE_M 180
```

Діагностична перевірка при введенні в експлуатацію (один раз):

```bash
py -3 .\tools\tune_cli.py --port COM12 --baud 115200 set FCGPS_FWD 1
```

Повернення у штатний режим:

```bash
py -3 .\tools\tune_cli.py --port COM12 --baud 115200 set FCGPS_FWD 0
```

## 6) Правила запису параметрів

- Mission Planner для параметрів фільтра - лише читання (Read/Refresh).
- Змінюйте параметри фільтра тільки через `tune_cli.py`.
- Після `set` зачекайте до 30-45 секунд, якщо `get` одразу повертає `No reply`.
- Після будь-яких змін параметрів перезавантажте STM32 перед польотом.

## 7) Типові тригери DR1

- no-fix / мало супутників (`sats < 5`),
- стрибок позиції (`SP_ABS_M`, `SP_JMP_MPS`),
- аномалія SNR (`SNR_*`, якщо увімкнено),
- аномалія висоти (`ALT_*`),
- невалідний EKF (`EKF_TRIPMS`).
